Bottom Navigation

A bottom navigation should be used only on mobiles when you have three to five destinations which are equally important. You only want to use them inside webapps. The Safari browser bar at the bottom on iPhones disappears when you scroll down and this flips the position of the bottom navigation.

A bottom navigation with 3 or less tabs will show all labels. Bottom navigation with 4 or more will hide the inactive labels.

place Nearby
account_balance_wallet Wallet
image Photos 
<nav class="m-bottom-navigation">
    <div class="m-bottom-navigation__tab">
        <a href="#" class="m-bottom-navigation__button">
            <i class="m-bottom-navigation__icon material-icons">place</i>
            <span class="m-bottom-navigation__label">Nearby</span>
        </a>
    </div>
</nav>

Initializing

To initialize the component use the following function with optional options.

$(selector).mBottomNavigation(options);
Options

The following options are available:

hideLabels: 4
The amount of tabs when the plugin should hide labels of inactive tabs. Material Documents say this needs to be 4.
onButtonClick: function(tab){}
When a button is clicked the function will be executed and will pass the clicked tab.
followNavigation: false
On tab click disable or enable the following of the href tag in the anchor tag
ripple: true
Show or hide the ripple effect on click.

Methods
$(selector).data("mBottomNavigation").select(index)
Select a tab by its index

    </div>
    <div class="m-bottom-navigation__tab">
        <a href="#" class="m-bottom-navigation__button m-bottom-navigation__button--active">

        </a>
    </div>
    <div class="m-bottom-navigation__tab">

And initialisation with JavaScript

$(".m-bottom-navigation").mBottomNavigation();
